================================================================================
Справка по базовым возможностям git bash
================================================================================

Иниицализировать пустой репозиторий
git init

Добавить файл или изменения файла
git add ИМЯ_ФАЙЛА

Добавить изменения всех файлов
git add .

Зафиксировать изменения (сохранить текущее состояние активной ветки)
git commit -m "СООБЩЕНИЕ_КОММИТА"

Связать локальный репозиторий с удалённым (добавить remote)
git remote add ИМЯ_REMOTE URL_РЕПОЗИТОРИЯ

Залить все изменнеия локальной ветки в ветку удалённого репозитория
git push -u ИМЯ_REMOTE ИМЯ_ВЕТКИ

Создать ветку
git branch ИМЯ_ВЕТКИ

Перейти в ветку
git checkout ИМЯ_ВЕТКИ

Удалить локальную ветку
git branch -d ИМЯ_ВЕТКИ

Удалить удалённую ветку
git push origin --delete ИМЯ_ВЕТКИ

Слить изменения из одной ветки в активную ветку
git merge ИМЯ_ВЕТКИ

Слить изменения из одной ветки в активную ветку с сохранением истории веток
git merge ИМЯ_ВЕТКИ --no-ff -m "СООБЩЕНИЕ_КОММИТА"
